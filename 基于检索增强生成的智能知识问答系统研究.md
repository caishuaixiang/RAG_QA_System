## 1. 题目

### 基于RAG的智能知识问答系统

采用 RAG 架构，目的是解决传统大模型"幻觉"问题。系统不是让 AI"凭记忆回答"，而是先"查资料"，再"写答案"，确保了回答的准确性、可解释性和可追溯性。

这种"检索 + 生成"的模式，也是当前大模型落地应用的主流方案。

系统可广泛应用于企业内部知识库问答、高校规章制度查询、产品技术支持、医疗指南辅助阅读等多个领域，具有较强的实用价值和推广前景。

## 2. 核心功能

构建一个通用型智能问答平台，能够对非结构化知识文档进行自动化处理，并支持自然语言提问与精准回答。主要功能模块如下：

| 功能模块 | 功能描述 |
|---------|---------|
| **知识文档管理** | 支持上传 PDF、Word、TXT 等格式的知识文档，按"知识域"分类管理（如企业制度、产品手册、校园规章等）。 |
| **文档解析与切片** | 利用文本解析技术提取文档内容，并按语义或段落进行合理切分，形成检索单元。 |
| **向量化与存储** | 调用嵌入（Embedding）模型将文本块转化为向量，存储至轻量级向量数据库，支持高效语义检索。 |
| **自然语言问答** | 用户输入自然语言问题，系统自动检索最相关知识片段，结合大语言模型生成准确、流畅的回答。 |
| **答案溯源机制** | 回答结果附带来源信息（文档名称、章节、相似度），增强可信度与可解释性。 |

## 4. 创新点

本系统在传统问答系统基础上，深度融合前沿 AI 技术，具备以下创新性：

### 1.采用 RAG 架构解决大模型"幻觉"问题

通过"检索 + 生成"双阶段机制，确保答案来源于真实知识文档，显著提升回答的准确性与可信度。

### 2.构建通用化、可扩展的知识问答框架

系统不依赖特定领域，支持多种知识类型（企业制度、产品说明、技术文档等），具备良好的泛化能力与迁移性。

### 3.轻量级实现方案适配毕业设计场景

采用"Java 主控 + 远程 API 调用 + 轻量向量库"技术路线，在保证功能完整的前提下，降低本地资源消耗与实现难度。

### 4.支持答案溯源与知识盲区分析

不仅返回答案，还标注来源文档；通过日志分析高频未命中问题，反向优化知识库建设，形成闭环。

## 5. 涉及智能算法

项目涉及的智能算法清单：

| 算法类别 | 具体算法/技术 | 在系统中的作用 |
|---------|-------------|--------------|
| **文本向量化算法** | Sentence Embedding（如 BGE、text2vec） | 将文本转换为语义向量，实现"语义理解" |
| **语义相似度计算** | 余弦相似度（Cosine Similarity） | 比较问题与知识块之间的语义接近程度 |
| **文本切片算法** | 固定长度切片、语义感知切片 | 将长文档拆分为更小的检索单元，以提高检索质量和效率 |
| **检索排序算法** | Top-K 相似度排序 | 根据相似度对候选结果进行排名，并返回最相关的 K 个文本块 |
| **提示工程（Prompt Engineering）** | 上下文增强、指令优化 | 通过精心设计提示来指导大模型生成更加准确和有用的回答或内容 |

---

### 重点算法详解

#### 1. 文本向量化算法（Embedding）

- **算法原理**：使用深度学习模型（如 BERT、BGE）将文本映射到高维向量空间，语义相近的文本向量距离更近。

- **在项目中**：
    - 文档切片 → 向量化 → 存入向量数据库
    - 用户提问 → 向量化 → 用于检索

#### 2. 语义相似度计算（余弦相似度）

- **算法原理**：

$$
\text{Cosine Similarity} = \frac{A \cdot B}{\|A\| \|B\|}
$$

值越接近 1，语义越相似。

- **在项目中**：
    - 向量数据库（Chroma/Milvus）内部使用余弦相似度进行检索
    - 系统返回 Top-3 最相似的知识块

#### 3. 文本切片算法（Text Chunking）

**算法原理**：将长文本拆分为固定长度或语义完整的片段

**实现方式**：

```java
public List splitByLength(String text, int size) {
    List chunks = new ArrayList&lt;&gt;();
    for (int i = 0; i &lt; text.length(); i += size) {
        int end = Math.min(i + size, text.length());
        chunks.add(text.substring(i, end));
    }
    return chunks;
}
#### 4. Top-K 检索排序算法

- **算法原理**：在向量空间中查找与查询向量最接近的 K 个向量

- **在项目中**：
  - 向量数据库自动实现
  - 可设置 `n_results=3` 返回前三名
